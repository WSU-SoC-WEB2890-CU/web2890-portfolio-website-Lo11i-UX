Fitness Tracker Web App

A JavaScript-based fitness tracker that lets users log workouts in the browser and see their progress over time.

Role: Front-End Developer & Designer Tools: HTML, CSS, JavaScript Course: WEB 2890 - JavaScript Project

Project Overview

This project is a browser-based fitness tracker built with vanilla JavaScript. The goal was to create a simple tool where users can log their workouts and quickly see what they've completed that day or week, without needing a backend or framework.

The app focuses on core JavaScript skills:working with forms, arrays of data, DOM manipulation, and basicstate management in the browser.

My Responsibilities

  • Planned the overall layout and interaction flow.
  • Designed the UI using HTML/CSS with a card-style layout.
  • Implemented all functionality in vanilla JavaScript.
  • Handled validation and user feedback for the workout form.
  • Made the layout responsive so it works on mobile and desktop.

Key Features

Workout logging

Users can add a new workout by filling out a form with fields such as activity name, duration, type, and intensity. When the form is submitted, JavaScript creates a workout object, stores it in an array, and renders it into the list on the page.

Dynamic workout list

All workouts are displayed in a table or card list that updates instantly when the user adds a new entry.

Summaries & totals

The tracker calculates simple summary information from the workouts array. This gave me practice looping through arrays and updating summary UI elements whenever the data changes.

Responsive UI

The UI is styled so that the form and workout list reflow nicely on smaller screens. I used flexible widths, stacking behavior, and media queries so users can interact easily from a phone or laptop.

JavaScript Implementation Details

  • Defined an array of workout objects and used functions to add new workouts and re-render the list.
  • Used event listeners for form submission and button clicks, preventing default behavior to handle everything in JavaScript.
  • Implemented basic validation with user-friendly error messages.
  • Cleaned and reset the form after each successful submission so users can quickly log multiple workouts.

What I Learned

This project helped solidify my understanding of how JavaScript can manage state entirely in the browser. I practiced:

  • Connecting forms to JavaScript logic using event listeners.
  • Transforming user input into structured data (objects).
  • Keeping the UI and data in sync by re-rendering based on the current state of the workouts array.
  • Thinking about user flows—how quickly someone can log an activity and visually see that it “counted.”
