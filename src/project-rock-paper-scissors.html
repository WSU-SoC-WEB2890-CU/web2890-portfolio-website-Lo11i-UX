Rock, Paper, Scissors Game

A browser-based Rock, Paper, Scissors game built with vanilla JavaScript that tracks wins, losses, and ties in a best-of-five match against the computer.

Role: Front-End Developer Tools: HTML, CSS, JavaScript Concepts: Game logic, DOM updates, state management

Project Overview

This project is a simple but fully interactive Rock, Paper, Scissors web game. Users play against the computer, see both choices, and watch the scoreboard update after each round.

My Responsibilities

  • Designed the layout for the scoreboard and button controls.
  • Wrote the full game logic in JavaScript, including win/lose/tie detection.
  • Connected UI elements (buttons, score fields, messages) to the logic using event listeners.

Gameplay & Features

Scoreboard with live stats

At the top of the interface, the scoreboard shows:

  • Total ties.
  • User wins and user losses.
  • Computer wins and computer losses.

Each value is stored in separate variables and updated after every round.

Resetting the game

A Reset button at the top calls reset(), which:

  • Resets all counters back to 0.
  • Clears the latest choices and round message.
  • Re-enables the choice buttons.
  • Updates the UI so the scoreboard returns to its starting state.

Project Details

  • Rock, Paper, Scissors game built with vanilla JavaScript.
  • Tracks wins, losses, and ties for both player and computer.
  • Implements "first to 5 wins" match logic with reset.
